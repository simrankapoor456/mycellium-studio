## Community settings

- Enable private vulnerability reporting if it is available for the repository.
- Keep Issues enabled for the checked-in bug and product-proposal forms.
- Do not add a question form until Discussions or another maintained support destination exists.
- Add branch protection only after required checks have stable workflow names; no workflow is introduced by Phase 7.1.
